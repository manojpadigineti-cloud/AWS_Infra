KUBERNETES – SCENARIO BASED (PRODUCTION)
========================

1. Q: EKS node disk utilization reached ~85%, SSH is restricted, and even after increasing EBS volume the OS didn’t reflect changes — how do you fix it without node access?
   A: Deploy a privileged pod with hostPath mount, chroot into host, run growpart + filesystem expansion (xfs_growfs/resize2fs). Used in production for zero-access node recovery.

2. Q: Why doesn’t EBS volume auto-expand at OS level after resizing?
   A: AWS expands block device only; partition + filesystem must be manually extended inside OS.

3. Q: Why use a privileged pod instead of SSH?
   A: Secure environments block SSH; privileged pod provides controlled, auditable access via Kubernetes.

4. Q: What risks exist with privileged pods?
   A: Full host access can compromise node; restrict via RBAC and use only for emergency ops.

5. Q: How do you ensure this doesn’t happen again?
   A: Use monitoring alerts (disk usage), automate node bootstrap scripts to auto-expand volumes.

6. Q: What tools are required for disk expansion in Linux?
   A: growpart for partition resizing; xfs_growfs or resize2fs for filesystem expansion.

7. Q: How do you identify correct device and partition?
   A: Use lsblk/df -h to map root filesystem to underlying device (e.g., /dev/xvda1).

8. Q: What is the difference between partition resize and filesystem resize?
   A: Partition resize increases disk allocation; filesystem resize makes OS usable space available.

9. Q: How would you automate this fix in production clusters?
   A: Use user-data scripts, DaemonSets, or AMI baking to auto-expand volumes on boot.

10. Q: How do you validate successful expansion?
    A: Run df -h and ensure new disk size reflects expected capacity.

   Shell Scripts
=====================

Prevent disk space issues by deleting old logs automatically.

#!/bin/bash
echo "Deleting log files older than 7 days..."
find /var/log -type f -name "*.log" -mtime +7 -exec rm -f {} \;

2. Check SSL Certificate Expiry 

Avoid downtime by monitoring SSL certificate expiration.

#!/bin/bash
DOMAIN="yourdomain.com"
echo | openssl s_client -servername $DOMAIN -connect $DOMAIN:443 2>/dev/null | openssl x509 -noout -enddate

3. Automate System Package Updates 

Keep your Linux system updated without manual intervention.

#!/bin/bash
echo "Updating system packages..."
sudo apt update && sudo apt upgrade -y

4. Send Alerts for High CPU Usage 

Get notified when CPU usage crosses a threshold.

#!/bin/bash
THRESHOLD=80
USAGE=$(top -bn1 | grep "Cpu(s)" | awk '{print $2 + $4}')
if (( $(echo "$USAGE > $THRESHOLD" | bc -l) )); then
 echo "High CPU Usage: $USAGE%" | mail -s "CPU Alert" you@example.com
fi

5. Restart a Failing Kubernetes Pod 

Automatically restart pods stuck in CrashLoopBackOff.

#!/bin/bash
echo "Restarting failed pods..."
kubectl get pods --all-namespaces | grep CrashLoopBackOff | awk '{print $2}' | xargs kubectl delete pod
