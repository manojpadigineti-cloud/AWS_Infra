[1] Design multi-region DR with minimal downtime. Approach?
"For multi-region DR, I use Route 53 with latency/health check routing to primary region. Deploy identical infrastructure across 2-3 regions using CloudFormation/Terraform. Enable cross-region RDS read replicas + automated failover with EventBridge. Use S3 CRR for data + Lambda for final sync. RTO <5min via Route 53 weighted routing switch."

[2] How do you secure cross-region S3 replication and validate data integrity?
"Enable S3 CRR with KMS encryption (customer-managed keys). Use bucket policies restricting replication role. Validate integrity with S3 Inventory + Athena queries comparing object counts/hashes. Enable versioning + MFA Delete. Monitor with CloudWatch Events → Lambda alerting on replication failures."

[3] How do you design hybrid routing between AWS and GCP?
"Use AWS Transit Gateway + GCP Cloud Router with BGP. Establish IPsec VPN or Dedicated Interconnect. Route 0.0.0.0/0 via Transit Gateway with AWS as primary. Health checks via VPC Reachability Analyzer. Failover via BGP AS path prepending + Cloud DNS health checks."

[4] EC2 app failing health checks behind ALB. How do you troubleshoot?
"First, SSH to EC2 → check app logs/service status (systemctl status). Verify ALB health check path returns 200 (curl -v). Check SG allows ALB traffic (port/protocol). Review CloudWatch metrics (HealthyHostCount, ResponseTime). Test locally: nc -l port. Restart service + scale if CPU>80%."

[5] RDS running out of storage. Immediate actions?
"Check CloudWatch FreeStorageSpace + TopQueries. Enable storage autoscaling (set max to 2x current). Run ANALYZE + DELETE old data via pgAdmin. Switch to larger instance (modify → apply immediately). Long-term: implement table partitioning + RDS Performance Insights."

[6] How do you reduce AWS cost for idle resources?

[7] S3 bucket accidentally deleted. How do you recover?
