[2] Difference between Terraform and configuration management tools?
Terraform is an Infrastructure as Code (IaC) provisioning tool that creates and manages infrastructure resources (VMs, VPCs, databases, Kubernetes clusters) using a declarative model, whereas configuration management tools (like Ansible, Chef, Puppet) focus on configuring software inside already provisioned infrastructure. In enterprise environments, Terraform is used for Day-0 provisioning (infrastructure lifecycle), while configuration management handles Day-1/Day-2 operations like patching, software installs, and OS configuration. Terraform is immutable (recreate strategy preferred), whereas config tools are mutable (in-place changes).

[5] What is Terraform state file?
Terraform state file (terraform.tfstate) is a JSON file that maintains a mapping between Terraform configuration and real-world infrastructure. It stores resource metadata, IDs, dependencies, and outputs. In enterprise setups, state is critical for idempotency, drift detection, and dependency graph resolution. It must be securely stored (remote backend like S3 with DynamoDB locking or Terraform Cloud) because it may contain sensitive data.

[7] What does terraform init do?
terraform init initializes a Terraform working directory by downloading provider plugins, initializing backend configuration (remote/local state), and preparing modules. In enterprise workflows, it also ensures version consistency using provider and module version constraints.

[8] What is terraform plan?
terraform plan performs a dry run and shows the execution plan (create/update/delete actions) required to reach the desired state. It helps teams validate changes before applying. In enterprise pipelines, plan output is reviewed and approved before apply (e.g., PR-based workflow).

[9] What is terraform apply?
terraform apply executes the changes defined in the plan and provisions/modifies infrastructure. It interacts with provider APIs and updates the state file. In enterprise setups, apply is controlled via CI/CD with approvals and role-based access.

[10] What are resources in Terraform?
Resources are the fundamental building blocks representing infrastructure components like aws_instance, azurerm_virtual_machine, or google_compute_instance. Each resource maps to an API object in the provider and is tracked in the state.

[11] What are variables in Terraform?
Variables are input parameters that allow dynamic and reusable configurations. They are defined using variable blocks and can be passed via CLI, tfvars, environment variables, or CI/CD pipelines. Enterprise usage includes environment-specific configurations.

[12] What are outputs in Terraform?
Outputs are values exported after execution, such as IP addresses or resource IDs. They are used for integration with other modules, pipelines, or tools (e.g., passing ALB DNS to deployment scripts).

[13] What are local variables in Terraform?
Local variables (locals block) are used for internal computations and to simplify expressions within a module. They improve readability and avoid duplication in enterprise-scale codebases.

[14] What are modules in Terraform? Do you use default modules or custom ones?
Modules are reusable, self-contained Terraform configurations. Enterprises typically use custom modules stored in private registries to enforce standards, security, and compliance. Public modules may be used but are often wrapped or forked for control.

[15] How do you manage secrets in Terraform?
Secrets should not be hardcoded. Enterprise approaches include using HashiCorp Vault, AWS Secrets Manager, Azure Key Vault, or GCP Secret Manager. Sensitive variables are marked with sensitive = true, and remote state encryption is enabled. Secrets are injected dynamically via CI/CD or data sources instead of storing them in state.

[16] What is terraform.tfvars?
terraform.tfvars is a file used to define variable values. It allows environment-specific configurations (dev, qa, prod). In enterprises, separate tfvars files are used per environment and managed via Git.

[40] Explain depends_on vs implicit dependency — when does Terraform get it wrong?
Implicit dependencies are inferred from resource references. depends_on is explicit. Terraform may get it wrong when dependencies are not directly referenced (e.g., null_resource, external scripts), requiring manual depends_on.
