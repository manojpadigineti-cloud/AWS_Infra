[1] In a Pod, how many containers can you run? Why use multiple containers?
A Pod can run one or more containers. Multiple containers are used for sidecar patterns (logging, proxy, monitoring), init containers, or tightly coupled processes sharing the same network and storage.

[2] What is a Rolling Update in Kubernetes?
Rolling Update gradually replaces old Pods with new ones without downtime, ensuring high availability.

[3] What is the Recreate strategy in Kubernetes?
Recreate deletes all existing Pods before creating new ones, causing downtime but ensuring clean deployment.

[4] How do you perform maintenance activity on a Kubernetes node?
Drain the node (kubectl drain), perform maintenance, then uncordon (kubectl uncordon).

[5] How do you control resource usage of a Pod?
Using requests and limits in Pod spec to define CPU and memory consumption.

[6] What are the units of CPU and memory in Pod definition?
CPU: millicores (m), Memory: bytes (Ki, Mi, Gi).

[7] How do you upgrade Kubernetes?
Upgrade control plane first, then worker nodes, using tools like kubeadm or managed services (EKS/GKE/AKS).

[8] Difference between Helm and Kubernetes Operator?
Helm is a package manager for deploying apps, while Operators are controllers that manage application lifecycle using CRDs.

[9] What are Kubernetes node components and their roles?
kubelet (agent), kube-proxy (networking), container runtime (Docker/containerd).

[10] What is a Pod Disruption Budget (PDB)?
Defines minimum available pods during voluntary disruptions to maintain availability.

[11] What are Kubernetes deployment strategies?
Rolling Update, Recreate, Blue-Green, Canary.

[12] Pods in one namespace work internally but not from another namespace. How do you troubleshoot?
Check DNS, network policies, service configuration, and namespace isolation rules.

[13] HPA not scaling despite high CPU usage. How do you debug?
Check metrics-server, resource requests, HPA configuration, and metrics availability.

[14] New nodes join EKS but pods are stuck in Pending. Possible reasons?
Insufficient resources, taints/tolerations mismatch, missing IAM roles, or node labels mismatch.

[15] How do you troubleshoot a Pod not getting scheduled?
Check events (kubectl describe pod), node resources, taints, affinity rules.

[16] How do you run a Pod on a specific node?
Use nodeSelector, node affinity, or tolerations.

[17] How do you colocate Pods for performance?
Use pod affinity rules to schedule pods on the same node.

[18] How do you provide persistent storage for Pods?
Using PersistentVolume (PV) and PersistentVolumeClaim (PVC).

[19] How do containers in a Pod share a single IP?
Pod has a single network namespace; all containers share it.

[20] How do you expose Kubernetes services externally?
Using NodePort, LoadBalancer, or Ingress.

[21] How does Pod-to-Pod communication work?
Via flat network model where each Pod has unique IP.

[22] How does Pod-to-Service communication work?
Through kube-proxy which routes traffic to backend Pods.

[23] How can you improve Kubernetes security?
RBAC, Network Policies, Pod Security Standards, secrets management, image scanning.

[24] How do you monitor Kubernetes?
Using Prometheus, Grafana, CloudWatch, or Azure Monitor.

[25] How do you centralize logs from Pods?
Using EFK/ELK stack (Elasticsearch, Fluentd, Kibana) or Loki.

Kubernetes uses a flat networking model where every Pod has a unique IP and can communicate without NAT. CNI plugins (Calico, Flannel, Cilium) implement this networking.

[61] Why is timestamp standardization critical in Kubernetes logging?
Ensures correlation across distributed systems for debugging and observability.

[62] What is the impact of stdout vs stderr in container logging?
Both are captured by container runtime, but stderr is typically used for errors and may be treated differently by log collectors.

[63] Why should you specify container name in kubectl logs?
Required when a Pod has multiple containers to fetch logs from a specific container.

[64] Do Kubernetes pods generate logs directly?
No, containers write logs to stdout/stderr; kubelet and runtime handle log collection.

[65] Why can kubectl logs be misleading during failures?
Logs may not show startup failures, previous crashes, or missing logs due to buffering.

[66] What are common non-application causes of pod failures?
Resource limits, node issues, networking, storage problems, or misconfiguration.

[67] Why is structured logging preferred in production systems?
It enables better parsing, searching, and analysis in centralized logging systems.

[68] When should you use plaintext vs structured logs?
Plaintext for simple debugging; structured logs (JSON) for production observability.

[69] What is the biggest advantage of structured logging?
Machine-readable format enabling efficient querying and correlation.

[70] Why is proper log level usage important in DevOps?
Helps reduce noise, improve debugging, and optimize storage/monitoring.

[71] How should log levels be used in real-world systems?
INFO for normal ops, WARN for potential issues, ERROR for failures, DEBUG for troubleshooting.

[72] What happens if log levels are misused?
Noise, missed critical alerts, increased costs, and difficult debugging.

[211] EKS node disk utilization reached ~85%, SSH is restricted, and even after increasing EBS volume the OS didn’t reflect changes — how do you fix it without node access?
Use a privileged DaemonSet or debug pod to exec into node namespace, then expand partition and filesystem using growpart and resize2fs.

[212] Why doesn’t EBS volume auto-expand at OS level after resizing?
Because resizing at cloud level only increases block device size; OS partition and filesystem must be manually expanded.

[213] Why use a privileged pod instead of SSH?

[214] What risks exist with privileged pods?
