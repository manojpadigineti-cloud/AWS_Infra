# Ingress annotation
alb.ingress.kubernetes.io/target-type: instance
type: NodePort

# Ingress annotation
alb.ingress.kubernetes.io/target-type: ip
type: ClusterIP

Based on annotation, service type should be given
---------------------------------------------------------------------------------

IngressClassParams          IngressClass
(the settings/config)  →→→  (the class that uses those settings)
        │                           │
        │                           ▼
        │                   Ingress resources
        └──────────────────→ reference this class

IngressClassParams = blueprint/config (what kind of ALB, what scheme)
IngressClass = the class name that ingresses use to pick a controller
Ingress = the actual route that references the class

Mandatory Annotations for alb
-----------------------------
alb.ingress.kubernetes.io/scheme: internet-facing
alb.ingress.kubernetes.io/target-type: ip  # or instance
kubernetes.io/ingress.class: alb -- which ingressclass to be used

        Value – 1
If you’re using the AWS Load Balancer Controller version 2.1.1 or earlier, subnets must be tagged as follows. If using version 2.1.2 or later, this tag is optional

Download an IAM policy for the AWS Load Balancer Controller that allows it to make calls to AWS APIs on your behalf.
curl -O https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/v2.14.1/docs/install/iam_policy.json
---
Create an IAM policy using the policy downloaded in the previous step.
aws iam create-policy \
    --policy-name AWSLoadBalancerControllerIAMPolicy \
    --policy-document file://iam_policy.json
---

### If this doesn't worked out, create a podidentity role with any name like lb_pod_identity and attach the above downloaded policy (AWSLoadBalancerControllerIAMPolicy), then create a service account in cluster within (kube-system) namespace with the name we like (should be same if we create it with terraform ex: service_account = "aws-load-balancer-controller") use the same name to create SA.

==========
TF Example -- This creates an access entry or if we want to create manually ==> AWS Console → EKS → Your Cluster → Access tab → Pod Identity Associations
==========
resource "aws_eks_pod_identity_association" "pod-identity-associate_aws_controller" {
  for_each = var.eks
  cluster_name    = each.value["name"]
  namespace       = "kube-system"
  service_account = "aws-load-balancer-controller"
  role_arn        = var.aws_controller_role[each.value["aws_controller_role"]]
}

Replace the values for cluster name, region code, and account ID
eksctl create iamserviceaccount \
    --cluster=prod-cluster \
    --namespace=kube-system \
    --name=aws-load-balancer-controller \
    --attach-policy-arn=arn:aws:iam::093468662840:policy/AWSLoadBalancerControllerIAMPolicy \
    --override-existing-serviceaccounts \
    --region us-east-1 \
    --approve

# alb-ingressclassparams.yaml - CORRECT API for LBC
apiVersion: elbv2.k8s.aws/v1beta1
kind: IngressClassParams
metadata:
  name: alb
spec:
  scheme: internet-facing

---
apiVersion: networking.k8s.io/v1
kind: IngressClass
metadata:
  name: alb
  annotations:
    ingressclass.kubernetes.io/is-default-class: "true"
spec:
  controller: ingress.k8s.aws/alb
  parameters:
    apiGroup: elbv2.k8s.aws
    kind: IngressClassParams
    name: alb
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: deployment-2048
spec:
  selector:
    matchLabels:
      app.kubernetes.io/name: app-2048
  replicas: 2
  template:
    metadata:
      labels:
        app.kubernetes.io/name: app-2048
    spec:
      containers:
        - image: public.ecr.aws/l6m2t8p7/docker-2048:latest
          imagePullPolicy: Always
          name: app-2048
          ports:
            - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: service-2048

  ports:
    - port: 80
      targetPort: 80
      protocol: TCP
  type: NodePort
  selector:
    app.kubernetes.io/name: app-2048

# 2048-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: ingress-2048
  annotations:
    kubernetes.io/ingress.class: alb
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTP": 80}]'
spec:
  ingressClassName: alb
  rules:
  - http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: service-2048
            port:
              number: 80
